Encrypted Chat Relay System

Secure Multi-Client Network Communication

A real-time encrypted chat relay system implemented in C that demonstrates secure network communication using socket programming. The system features a central server that broadcasts messages between multiple clients using AES-256-CBC encryption to ensure message confidentiality across the network.

How the Relay System Works

Server Architecture

The server acts as a central relay hub, listening on port 3490 for incoming client connections. It maintains a list of connected clients (up to 10) and broadcasts each message to all other connected clients, creating a group chat environment.

Client Communication

Each client connects to the server and can send messages that are timestamped and encrypted before being relayed to other clients. The client application handles both sending user input and receiving/decrypting messages from other users in real-time.

Encryption Protocol

Messages are encrypted using AES-256-CBC (Advanced Encryption Standard with 256-bit keys in Cipher Block Chaining mode) via OpenSSL. Each message is encrypted on the server before broadcast, ensuring that data transmitted over the network remains confidential.

How to Compile

Prerequisites

The system requires a Unix-like environment (Linux, macOS, or Windows with WSL) and the following:

  • GCC compiler
  • OpenSSL development libraries (libssl-dev)
  • POSIX socket support

Compilation Commands

Compile the server:

gcc -o server server.c -lssl -lcrypto -Wall

Compile the client:

gcc -o client client.c -Wall

Automated Scripts (Windows)

PowerShell scripts are provided for automatic compilation and execution:

  • start_server.ps1 - Compiles and starts the server
  • start_client.ps1 [hostname] - Compiles and connects a client

How to Operate

Starting the Server

Using PowerShell script:

.\start_server.ps1

Manual execution:

./server

The server will listen on port 3490 and wait for client connections.

Connecting Clients

Using PowerShell script (local):

.\start_client.ps1 localhost

Using PowerShell script (remote):

.\start_client.ps1 192.168.1.100

Manual execution:

./client [server_hostname]

Using the Chat System

  1. Start the server first in one terminal
  2. Launch one or more clients in separate terminals
  3. Type messages in any client and press Enter
  4. Messages appear in all other connected clients
  5. Type quit to disconnect a client
  6. Press Ctrl+C to stop the server

Testing Scenarios

Local Testing (Same Machine)

Open two terminals:

  • Terminal 1: .\start_server.ps1
  • Terminal 2: .\start_client.ps1 localhost

Network Testing (Multiple Machines)

On server machine:

  1. Start the server: .\start_server.ps1
  2. Note the IP address: ipconfig (Windows) or ifconfig (Linux)

On client machine(s):

.\start_client.ps1 192.168.1.XXX

Replace XXX with the server's IP address.

Technical Implementation

The system is implemented in C using standard POSIX sockets for network communication and OpenSSL for cryptographic operations. The server uses a broadcast architecture where each incoming message is encrypted and relayed to all connected clients except the sender. The source code demonstrates practical network programming concepts including socket creation, connection handling, select-based I/O multiplexing, and secure encryption practices.
